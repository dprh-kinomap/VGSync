Umfassende Git-Anleitung (Deutsch)

1. Was ist Git?

Git ist ein Versionskontrollsystem, mit dem du Änderungen an deinen Dateien (meist Quellcode) im Zeitverlauf speichern und verwalten kannst. Es erlaubt dir, ältere Zustände wiederherzustellen, parallel an verschiedenen Features zu arbeiten und Änderungen mit anderen zu teilen.

2. Git installieren

  • Windows: Git für Windows herunterladen und installieren.
  • macOS: brew install git (oder via offizieller Download)
  • Linux (z.B. Ubuntu): sudo apt-get install git

Prüfe die Installation durch:

git --version

3. Neues Git-Repository anlegen

  1. In ein bestehendes Projekt wechseln: 
    cd /pfad/zu/deinem/projekt
  2. Git initialisieren: 
    git init
    Dadurch entsteht ein versteckter Ordner .git, der alle Versionsinformationen speichert.
  3. .gitignore anlegen (empfohlen):

    Erstelle eine Datei namens .gitignore, in der du Dateien/Verzeichnisse definierst, die nicht versioniert werden sollen (z.B. venv/, node_modules/, *.log etc.).

4. Änderungen festhalten (Status, Add, Commit)

  • Status prüfen: 
    git status
    Zeigt, welche Dateien verändert wurden und ob sie bereits zum Commit vorgemerkt sind.
  • Änderungen zum Commit vormerken (stagen): 
    git add datei.txt
git add .
    git add . fügt alle geänderten Dateien hinzu.
  • Commit ausführen (speichern): 
    git commit -m "Sinnvolle Beschreibung der Änderung"

Jeder Commit ist ein Schnappschuss deines Projekts. Achte auf aussagekräftige Commit-Nachrichten!

5. Unterschiede anzeigen (Diff)

Wenn du sehen möchtest, was sich seit dem letzten Commit geändert hat:

git diff

Um den Unterschied zwischen Staging und Commits zu sehen:

git diff --staged

6. Alle gespeicherten Versionen anzeigen (Log)

Mit git log siehst du alle Commits in chronologischer Reihenfolge:

git log

Etwas kompakter:

git log --oneline --graph --decorate

Damit siehst du eine übersichtliche Liste mit Verzweigungen (Branches) und Commit-Nachrichten.

7. Branches (Zweige) nutzen

Branches erlauben parallele Entwicklungszweige. Standardmäßig hast du den Branch main (oder master).

  • Aktuelle Branches anzeigen: 
    git branch
  • Neuen Branch erstellen & wechseln: 
    git checkout -b feature-xyz
  • Zwischen Branches wechseln: 
    git checkout main
  • Branch mergen (zusammenführen): 
    git checkout main
git merge feature-xyz
    Jetzt sind die Änderungen aus feature-xyz in main.
  • Branch löschen: 
    git branch -d feature-xyz

Möchtest du nur einzelne Dateien aus einem Branch übernehmen, kannst du die Zusammenführung händisch lösen (Stichwort cherry-pick), siehe unten.

8. Einzelne Commits übernehmen (Cherry-Pick)

Angenommen, du hast im Branch feature-xyz einzelne Commits, die du in main integrieren willst, ohne den gesamten Branch zu mergen. Dafür nutzt du cherry-pick.

  1. Finde den Commit-Hash, den du willst (z.B. mit git log --oneline).
  2. Wechsle in den Ziel-Branch, z.B. main: 
    git checkout main
  3. Cherry-Pick aus feature-xyz: 
    git cherry-pick COMMIT-HASH

So übernimmst du nur diesen einen Commit.

9. Änderungen zurücksetzen

Es gibt verschiedene Möglichkeiten, Änderungen rückgängig zu machen:

  • Unstagen (Datei aus Staging entfernen): 
    git restore --staged datei.txt
  • Datei auf letzten Commit zurücksetzen: 
    git checkout -- datei.txt
  • Letzten Commit rückgängig machen (neuen Commit erzeugen): 
    git revert HEAD
    Erstellt einen neuen Commit, der die Änderungen von HEAD rückgängig macht.
  • Auf älteren Commit zurücksetzen (History überschreiben): 
    git reset --hard COMMIT-HASH
    Achtung: Überschreibt den Verlauf lokal; in Teamprojekten eher revert benutzen!

10. Änderungen zwischenparken (Stash)

Mit stash kannst du aktuelle Änderungen „parken“, ohne sie zu committen:

git stash

Um sie wiederherzustellen:

git stash pop

Liste aller Stashes:

git stash list

11. Tags (Versionen markieren)

Um einen bestimmten Commit als Version (z.B. v1.0) zu markieren:

git tag v1.0

Alle Tags anzeigen:

git tag

Auf einen Tag zurückspringen (detached HEAD):

git checkout v1.0

12. GitHub (Remote-Repository)

GitHub ist ein Online-Dienst, der Git-Repositories hostet. Hier kannst du deinen Code hochladen, sichern und mit anderen teilen.

  1. Erstelle ein neues Repository auf GitHub.
  2. Verbinde dein lokales Repository damit: 
    git remote add origin https://github.com/DEIN_BENUTZERNAME/DEIN_REPOSITORY.git
git push -u origin main

Danach kannst du mit git push neue Commits hochladen und mit git pull Änderungen holen.

13. Push, Pull & Clone

  • Push (lokale Änderungen hochladen): 
    git push
  • Pull (Änderungen vom Remote holen und mergen): 
    git pull
  • Clone (bestehendes Repo herunterladen): 
    git clone https://github.com/DEIN_BENUTZERNAME/DEIN_REPO.git

14. Tipps & Best Practices

  • Regelmäßige, kleine Commits: Lieber oft committen, statt riesige Commits.
  • Commit-Messages: Klar und beschreibend (z.B. „Fix: Speicherleck behoben“)
  • Branches: Für neue Features oder Experimente immer separate Branches verwenden.
  • .gitignore aktuell halten: Keine unnötigen Dateien (Logs, Temp-Files, etc.) ins Repo.
  • Reset vs. Revert: In Teamprojekten lieber revert statt reset, um keine gemeinsame History zu überschreiben.
