1. Git installieren

Windows: git-scm.com/download/win

macOS:
brew install git
oder git-scm.com/download/mac

Linux (Ubuntu/Debian):
sudo apt-get update
sudo apt-get install git

Installationsprüfung:

git --version

2. Python-Projekt zu einem Git-Projekt machen

  1. In Projektverzeichnis wechseln
    cd /pfad/zu/deinem/python-projekt
  2. Git initialisieren
    git init
  3. .gitignore erstellen (empfohlen)
    Beispiel-Inhalt:
    __pycache__/
*.py[cod]
venv/
.venv/
build/
dist/
*.egg-info/
  4. Ersten Commit vorbereiten
    git add .
  5. Ersten Commit ausführen
    git commit -m "Initial commit"

3. Änderungen versionieren (Commits)

  1. Status prüfen
    git status
  2. Dateien zum Commit hinzufügen
    git add <datei>
    oder
    git add .
  3. Commit ausführen
    git commit -m "Beschreibender Commit-Text"

4. Alle Versionen anzeigen (Git Log)

git log

Mit q verlässt du den Log-Modus.

Alternativ kompakter:

git log --oneline --graph --decorate

5. Ältere Versionen (Commits) wiederherstellen oder ansehen

  1. Nur ansehen („detached HEAD“)
    git checkout <commit-hash>
    Zurück zum Hauptbranch:
    git checkout main
  2. Dauerhaft zurücksetzen
    • Revert (empfohlen):
      git revert <commit-hash>
    • Reset (Vorsicht):
      git reset --hard <commit-hash>

6. Arbeiten mit Branches

  • Neuen Branch erstellen & wechseln
    git checkout -b feature/neues_feature
  • Zurück zu main
    git checkout main
  • Branch zusammenführen (Merge)
    git checkout main
    git merge feature/neues_feature

7. Projekt auf GitHub hochladen

  1. Neues Repository auf GitHub anlegen
    (z.B. auf github.com einloggen und auf "New" klicken).
  2. Lokales Projekt mit GitHub Repo verknüpfen
    git remote add origin https://github.com/<benutzer>/<repo>.git
    git push -u origin main
  3. Zukünftige Änderungen hochladen
    git push

8. Weitere nützliche Git-Kommandos

  • Unterschiede anzeigen:
    git diff
  • Aktualisieren (fetch + merge):
    git pull
  • Stash (Änderungen zwischenparken):
    git stash
    git stash pop
  • Tags (Versionen markieren):
    git tag v1.0.0
    git push origin --tags

9. Zusammenfassung / Ablauf im Alltag

  1. Aktuellsten Stand holen: git pull
  2. Änderungen machen
  3. Status prüfen: git status
  4. Änderungen stagen: git add .
  5. Committen: git commit -m "Beschreibung"
  6. Pushen: git push

10. Tipps und Best Practices

  • Aussagekräftige Commit-Nachrichten schreiben.
  • Regelmäßige, kleine Commits sind besser als seltene, große.
  • .gitignore aktuell halten, damit keine unnötigen Dateien (z.B. venv) im Repo landen.
  • Branches für neue Features und Bugfixes nutzen, um den Hauptbranch stabil zu halten.
